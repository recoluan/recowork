# Scenarios

Use this template when a project needs planning, technical assessment, decision tracking, and durable AI collaboration.

## Planning And Solution Design

Use the generated `workspace/` directory to separate formal outputs from thinking traces:

- Put project background, goals, scope, and current status in `project-brief.md`.
- Put requirements, user scenarios, constraints, and clarifications in `01-requirements-and-constraints/`.
- Put tradeoff analysis, feasibility, architecture, and technical plans in `02-solution-design/`.
- Put phases, milestones, delivery plans, and durable decisions in `03-plan-and-decisions/`.
- Put brainstorming, intermediate reasoning, and discussion summaries in `04-thinking-traces/`.
- Put review, validation, and acceptance notes in `05-review-and-validation/`.
- Track questions that require user confirmation in `open-questions.md`.

## Rule

Do not collapse everything into one large document. Keep `index.md` files updated so humans and AI agents can read only what is needed.
