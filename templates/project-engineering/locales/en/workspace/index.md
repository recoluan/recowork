# Workspace Index

This is the entry index for the project workspace. Keep it short and use it for navigation.

## Key Documents

- `project-brief.md`: project background, problem, goals, scope, non-goals, current status, and constraints.
- `open-questions.md`: questions that need user input or block decisions.

## Sections

- `01-requirements-and-constraints/`: requirements, user scenarios, constraints, and clarifications.
- `02-solution-design/`: tradeoff analysis, feasibility, architecture, and implementation approach.
- `03-plan-and-decisions/`: phases, milestones, execution plans, and durable decisions.
- `04-thinking-traces/`: brainstorming, intermediate reasoning, and process notes.
- `05-review-and-validation/`: review, validation, acceptance, and follow-up checks.

## Usage

- Keep this index short.
- Add a one-line summary when an important document is created.
- Link to detailed documents instead of duplicating their content here.
