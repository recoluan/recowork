# 文档规范

## 结论在前

学习空间的文档既是课程记录，也是学习者能力与证据的长期载体。除 `index.md` 外，每份新建或实质更新的学习文档都应使用统一五段式，并且只聚焦一个学习问题、单元或结论。每次学习先从 `学习空间/index.md` 获取导航，再按需读取课程路线、进度和相关单元；学习结束后同步更新受影响索引。

## 正文

### 文档统一结构

除 `index.md` 外，每份实质性学习文档按以下顺序组织：

1. **元信息头**：标题、版本、日期、状态（草稿 / 评审中 / 当前有效 / 已替代 / 已归档）；状态的生命周期语义见 [产物保鲜规范](./产物保鲜规范.md)。
2. **结论在前**：用 3-5 句话说明已学会什么、证据、关键误区、限制或下一步。
3. **正文**：按层级组织概念、实验、练习、反馈、验证结果和学习者自己的解释。
4. **关联引用**：引用学习空间文件时使用相对路径 Markdown 链接。
5. **变更记录**：记录重要更新的日期、版本和摘要。

### 文档粒度与索引

- 一份文档聚焦一个学习单元、一个问题、一次复盘或一个可复用结论；主题变化时拆分。
- 不把整门课程、所有练习和全部复盘堆在同一份正文中。
- `index.md` 只保留元信息、目录条目、一句话简介、状态和更新时间，不重复课程正文或过程细节。

### 按需获取与回写

1. 先读 `学习空间/index.md`，再按需读学习简报、课程路线、学习进度和当前目录的 `index.md`。
2. 只打开本次学习需要的单元、练习和复盘，不预加载整个学习空间。
3. 新建、实质更新、定稿或废弃文档后，更新父级 `index.md` 的相对链接、简介、状态和更新时间。
4. 每次学习结束时，检查索引是否能让下一次学习快速恢复上下文。
