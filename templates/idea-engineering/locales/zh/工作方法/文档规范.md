# 文档规范

## 结论在前

想法探索既要保留发散过程，也必须让当前结论可定位、可复用。除 `index.md` 和极短临时清单外，每份实质性文档采用统一五段式且只聚焦一个主题。开始探索时先读 `想法空间/index.md`，再按需打开具体文件；结束时更新受影响的索引。

## 正文

### 文档统一结构

除 `index.md` 外，实质性文档按以下顺序组织：

1. **元信息头**：标题、版本、日期、状态（草稿 / 评审中 / 当前有效 / 已替代 / 已归档）。
2. **结论在前**：用 3-5 句话说明当前结论、限制或下一步。
3. **正文**：用标题层级组织问题、方向、假设、证据或决策，并区分事实、推断和待确认事项。
4. **关联引用**：引用其他文件时使用相对路径 Markdown 链接。
5. **变更记录**：记录重要变更的日期、版本和摘要。

### 文档粒度与索引

- 一个文档只回答一个可命名的问题；主题变化时新建文档并相互链接。
- `index.md` 只做导航：保留链接、一句话简介、状态和更新时间，不复制正文。

### 按需获取与回写

1. 先读 `想法空间/index.md`，再按需读取想法简报、待确认问题和当前阶段的 `index.md`。
2. 只加载完成当前探索所需的文件，不默认读取整个想法空间。
3. 新建、实质更新、定稿、替代或归档文档后，更新父级 `index.md` 的链接、简介、状态和时间。
4. 探索结束时，确认下一位读者或 Agent 能从索引找到当前方向和下一步。
