开发中

新增与变更

  • 新增网页设计规范模板：本地 Agent 生成一份可复用规范，聊天 / 手机端提供可独立复制的响应式网页大 Prompt。
  • 本地初始化会安全集成到已有根目录 AGENTS.md，仅管理 RecoWork 标记区块；升级会保留用户修改或删除过的区块。
  • 工作流要求 AI 客观指出风险与替代方案，站点也将工作流模板和设计规范分开展示。
  • 站点试用在线加载的 Noto Sans SC 与 JetBrains Mono，统一中英文与代码字体；暂不写入本地字体资产。
  • 站点大标题改用平衡换行并使用完整区块宽度，避免字体调整后出现生硬的词组截断。

兼容性

已有根目录 AGENTS.md 在 RecoWork 受管区块之外的内容不会被改写；已有工作流不会被自动修改。

2026-07-21

1.0.0

已发布

新增与变更

  • RecoWork 收敛为聊天 / 手机端与电脑端 AI 助手两种使用环境。
  • 本地环境生成 AGENTS.md、工作方法和工作空间；聊天环境只生成启动、执行与续接/迁移三件套。
  • 长期结论写入工作空间的权威文档与索引，初始化不会覆盖已有 RecoWork 工作流。

破坏性变更与迁移

品牌专属 target、原生 skill、平台配置目录和 Notion/飞书导出已移除。旧命令保留别名；旧 Chat 工作流执行 rw status 或 rw upgrade 时会得到创建独立本地目录的迁移指引。

2026-07-20

0.4.0

已发布

新增

  • 学习模板会先形成学习约定并等待确认，再生成课程路线、章节、练习或项目方案。
  • 项目模板会先形成项目约定并等待确认，再生成完整方案、计划或实施改动。
  • npm 页面现在展示仓库、官网和问题反馈链接；CLI 说明增加完整文档与 AI 初始化入口。

兼容性

已有工作流不会自动改动。可运行 rw upgrade --plan . 查看更新；新初始化仅在大范围学习或项目设计前等待确认，已确认的局部任务可直接推进。

2026-07-12

0.3.0

已发布

新增

  • Target 按 locale 输出用户可见路径和静态内容。
  • 三套工作流加入文档规范、索引优先读取和索引维护。
  • 贡献者和 AI 智能体采用 Conventional Commits。
  • 加入工作流升级顾问：版本化清单、只读检查、安全更新和工作空间升级报告。

兼容性

旧项目需先执行 rw upgrade --adopt . 建立升级基线。升级时，已有工作空间文件始终保留；新增模板须显式使用 --scope workspace --add-missing，报告写入 .recowork/upgrade-reports/。
