工作理念

Agent runtime 负责执行，RecoWork 负责让工作有据可依。The agent runtime executes.RecoWork makes the work grounded.

RecoWork 不替代模型、工具调用、记忆引擎或任务调度。它是面向人和 AI Agent 的工作标准与项目事实层。

两个层次

能行动的 Agent，不等于知道该如何持续完成工作。

Agent runtime

模型、工具、文件访问与执行能力。

它决定 Agent 能做什么操作。

RecoWork

任务协议、专业标准、项目事实、验收与续接。

它决定一项工作要如何被定义、检查、保存和继续。

四项工作资产

探索可以发散，当前事实必须收敛。

任务

清楚说明结果、边界和下一步。

标准

让 Agent 遵守专业约束、确认规则和质量门槛。

项目事实

用当前有效的简报、决定和问题替代散乱聊天记录。

续接

让下一次工作知道从哪里继续，以及哪些仍未确认。

为什么分两种环境

低门槛入口与持续项目，需要不同的承诺。

Chat / Mobile 是入口

它提供启动、执行协议和可复制的续接摘要。连续性由用户保存摘要来维持，不模拟本地目录或自动记忆。

本地可执行 Agent 是核心承载环境

它可在本地项目中维护规范、工作空间、中间产物、索引、状态检查与安全升级。项目事实保存在用户选择的位置。

刻意不做

清楚的能力边界，才能让工作方式可信。

  • 不把 RecoWork 描述为又一个 Agent 平台，也不托管模型或 API Key。
  • 不承诺自动记忆、后台无限运行或无人审查的全自动完成。
  • 不把 Chat / Mobile 伪装成本地项目工作流的等价版本。
  • 不要求用户接受 AI 的建议；Agent 必须说明风险、不确定性与可信替代方案。
RecoWork面向人和 AI Agent 的工作标准与项目事实层。
