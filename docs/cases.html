案例

不同场景下，RecoWork 生成不同工作系统。

这些案例展示 template、target 和 locale 如何组合成可执行资产，而不是只生成一段提示词。

怎么读案例

每个案例都看三件事：场景、生成物、持续使用方式。

场景

用户到底在什么环境里工作，是手机聊天、本地项目，还是文档协作。

生成物

RecoWork 会生成哪些文件、Prompt、工作空间、规则和检查清单。

持续方式

下一次对话或下一次项目迭代，AI 如何从已有事实继续工作。

典型案例

从聊天、项目到学习，看看 RecoWork 如何落到真实工作里。

general-ai-workflowchat-mobilezh

手机里快速启动一个任务

用户在 ChatGPT 或 Kimi 中做方案整理、沟通稿、会议总结时，RecoWork 提供启动指令、任务执行协议和需要手动保存的续接摘要。任务变复杂时，摘要会整理为可迁移到本地 Agent 的交接包。

看到的结果：启动指令、任务执行协议、续接/迁移摘要；不创建本地工作空间。
  • AI 先澄清目标和完成标准。
  • 每轮对话结束留下可复制的记忆卡。
project-engineeringlocal-agent-projectzh

给一个新代码项目建立 AI 协作协议

电脑端 AI 助手初始化后得到 AGENTS.md、工作方法和工作空间。AI 开始工作前先读角色设定和文档规范，再按需读取索引。

看到的结果：AGENTS.md、工作方法、工作空间、质量检查与安全升级记录。
  • AI 先澄清并等待用户确认项目约定，再生成方案或实施改动。
  • 决策、风险和验证结果不会只留在聊天里。
  • 下一次协作可以从已有事实继续。
learning-engineeringlocal-agent-projecten

把学习计划变成可验证的课程路径

学习者想系统学习 LangChain、产品设计或编程主题时，RecoWork 生成学习简报、课程路线、进度追踪、练习和复盘空间。

看到的结果：学习简报、课程路线、进度追踪、练习区、问题复盘。
  • 导师先诊断基础和目标，并等待学习者确认学习约定。
  • 每个单元都要经过练习和反馈。
project-engineeringlocal-agent-projecten

跨本地 Agent 复用同一套项目约定

本地通用 target 只生成 AGENTS.md 和工作空间，不把相同方法复制为各平台专属 skill 或配置。

看到的结果：可在多个本地 Agent 间复用的 AGENTS.md、工作方法和工作空间。
  • 同一套方法不再被品牌配置分裂。
  • 工作空间始终归用户所有。
