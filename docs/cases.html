案例

不同场景下，RecoWork 生成不同工作系统。

这些案例展示 template、target 和 locale 如何组合成可执行资产，而不是只生成一段提示词。

怎么读案例

每个案例都看三件事：场景、生成物、持续使用方式。

场景

用户到底在什么环境里工作，是手机聊天、本地项目，还是文档协作。

生成物

RecoWork 会生成哪些文件、Prompt、工作空间、规则和检查清单。

持续方式

下一次对话或下一次项目迭代，AI 如何从已有事实继续工作。

典型案例

从聊天、项目到学习，看看 RecoWork 如何落到真实工作里。

general-ai-workflowchatgpt-chatzh

手机里持续推进一个复杂任务

用户在 ChatGPT 或 Kimi 中做方案整理、沟通稿、会议总结时，RecoWork 生成任务简报、待确认问题、检查清单和续聊记忆。

看到的结果：初始化 Prompt、任务 Prompt、续聊记忆卡、复用检查清单。
  • AI 先澄清目标和完成标准。
  • 每轮对话结束留下可复制的记忆卡。
project-engineeringcodex-projectzh

给一个新代码项目建立 AI 协作协议

Codex 初始化后得到 AGENTS.md、工作方法、工作空间和知识库。AI 开始工作前先读角色设定和文档规范，再按需读取索引。

看到的结果：AGENTS.md、工作方法、工作空间、知识库、质量检查 skill。
  • AI 先澄清并等待用户确认项目约定，再生成方案或实施改动。
  • 决策、风险和验证结果不会只留在聊天里。
  • 下一次协作可以从已有事实继续。
learning-engineeringnotion-workspaceen

把学习计划变成可验证的课程路径

学习者想系统学习 LangChain、产品设计或编程主题时，RecoWork 生成学习简报、课程路线、进度追踪、练习和复盘空间。

看到的结果：学习简报、课程路线、进度追踪、练习区、问题复盘。
  • 导师先诊断基础和目标，并等待学习者确认学习约定。
  • 每个单元都要经过练习和反馈。
project-engineeringclaude-code-projecten

为 Claude Code 项目生成真实约定文件

Claude Code target 会生成 CLAUDE.md 和 .claude/skills，不会复用 Codex 的路径，也不会把聊天 prompt 当成项目配置。

看到的结果：CLAUDE.md、.claude/skills/knowledge-capture、.claude/skills/self-review。
  • target 遵守真实工具约定。
  • 模板方法和工具输出规则独立演进。
