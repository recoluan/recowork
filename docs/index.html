让 AI 用工程化的方式，聪明地工作。 Make AI work smarter.Engineer the workflow.

RecoWork 是 AI 工作 harness：把上下文、规则、工具约定和检查机制沉淀成模板，再按真实工具环境生成 Prompt、项目规则、知识空间和续聊机制。

为什么需要

AI 很强，但多数使用方式并不稳定。

现在常见的问题

每次都重新解释背景、标准和偏好。

  • Prompt 越写越长，但质量仍然不稳定。
  • 聊天结束后，决策和经验很难沉淀。
  • 换工具后，原来的工作方式又要重搭。
RecoWork 的做法

把工作方式初始化成可复用的 AI 工作 harness。

  • 用模板定义角色、流程、检查和记忆。
  • 按 target 生成真实工具能用的文件或 Prompt。
  • 用 locale 控制语言和用户可见命名。

适合谁

聊天快速开始，本地 Agent 持续推进。

日常 AI 用户

在 ChatGPT、Claude、Kimi 或豆包里做写作、总结、计划、分析，用续接摘要手动衔接下一次对话；需要长期沉淀时迁移到本地 Agent。

如何工作

三个选择，生成可用的 AI 工作资产。

01Template

选择工作流模板，或选择要跨项目复用的网页设计规范。

02Target

选择真实使用位置：聊天窗口、本地项目、文档工作区。

03Locale

选择输出语言和用户可见命名，保留工具约定文件名。

04Use

通过 AI Prompt 或 CLI 初始化，然后在实际工具里继续协作。

内置能力

先选工作方式，再按需要附加规范。

工作流模板定义 AI 如何澄清、推进、复盘和续接一类工作。

设计规范可跨项目复用，用来约束 AI 的视觉、响应式、状态和自检质量。

idea

idea-engineering

先发散脑暴，再用方向归纳、假设和小验证收敛到值得推进的下一步。

project

project-engineering

先澄清并确认项目约定，再建立 AGENTS.md、文档规范、质量门禁和持续沉淀机制。

learning

learning-engineering

把学习拆成诊断、课程路线、章节练习、项目实践、反馈、复盘和知识沉淀。

web design

web-design-standard

为 AI 生成或改造 HTML / Web 页面提供一份可复用规范，覆盖 token、响应式、组件状态和可访问性自检。
